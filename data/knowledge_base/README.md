# Knowledge Base

放在這個目錄的 `.txt` 和 `.md` 檔案，會在執行 `python scripts/init_knowledge_base.py` 時被 chunk、embed，存進 PostgreSQL `knowledge_chunks` 表，供 RAG agent 向量搜尋。

> 此目錄的 `.txt` 和 `.md` 已加入 `.gitignore`，不會被 commit。

---

## 如何新增知識

1. 把檔案放進這個目錄（`.txt` 或 `.md` 皆可）
2. 確認 PostgreSQL 已啟動：`docker compose up -d postgres`
3. 執行 ingest：`python scripts/init_knowledge_base.py`
4. 同一份文件重複執行不會重複寫入（以檔案路徑為 `doc_id` 做去重）

---

## 適合放什麼

- 技術分析筆記（均線策略、KD 操作方式、量價關係）
- 產業研究摘要
- 個人交易規則與停損邏輯
- 任何希望 agent 能引用的背景知識

---

## Chunking 策略

目前採三層降級：

```
1. 優先按空行（\n\n）切成段落
       ↓ 段落 ≤ 500 字
   直接使用整段（保留完整語意）

       ↓ 段落 > 500 字
2. 按句號／？／！切句子，累積至接近 500 字再斷
   （不會切在句子中間）

       ↓ 單句 > 500 字（極少見）
3. 按字數硬切，overlap 50 字
```

**建議的文件格式：**

- 每個知識點之間空一行，讓每條策略成為獨立 chunk
- 不需要特定格式，純文字條列或 Markdown 皆可

**範例（好切）：**
```
當投信連續站在買方，跌破月均線後重新站上，可買進。

一波大漲後的跌深反彈，等站上月均線後可買進搶反彈。
沿五日十日往上走，守住十日有機會挑戰前高。
```

**範例（不好切）：**
```
當投信連續站在買方，跌破月均線後重新站上，可買進。一波大漲後的跌深反彈，等站上月均線後可買進搶反彈。沿五日十日往上走，守住十日有機會挑戰前高。（全部擠在一行，無法按段落切）
```
